# webpack 性能优化

# resolve 解析

精简 resolve 配置，让 webpack 在查询模块路径时尽可能快速地定位到需要的模块，不做额外的查询工作，那么 webpack 的构建速度也会快一些

resolve: {
  modules: [
    path.resolve(__dirname, 'node_modules'), // 使用绝对路径指定 node_modules，不做过多查询
  ],

  // 删除不必要的后缀自动补全，少了文件后缀的自动匹配，即减少了文件路径查询的工作
  // 其他文件可以在编码时指定后缀，如 import('./index.scss')
  extensions: [".js"],

  // 避免新增默认文件，编码时使用详细的文件路径，代码会更容易解读，也有益于提高构建速度
  mainFiles: ['index'],
}

# loader 解析

我们在使用 loader 的时候，尽可能把 loader 应用的文件范围缩小，例如 node_modules 目录下的其他依赖类库文件，基本就是直接编译好可用的代码，无须再经过 loader 处理了

rules: [
  {
    test: /\.js?/,
    include: [
      path.resolve(__dirname, 'src'),
      // 限定只在 src 目录下的 js/jsx 文件需要经 babel-loader 处理
      // 通常我们需要 loader 处理的文件都是存放在 src 目录
    ],
    use: 'babel-loader',
  },
  // ...
],

# Tree Shaking

Tree Shaking 本质就是将引用到的代码进行一次整合打包。

  • 在webpack中 使用optimization中的usedExports字段；
//webpack.config.js
...
 optimization:{
    usedExports:true
}
...

在没有开启的情况，如下图： 在开启的情况，如下图

并且文件的大小对比如下

usedExports:false;
// asset bundle.js 3.53 KiB [emitted] (name: main);

usedExports:true;
// asset bundle.js 2.76 KiB [emitted] (name: main);

# Minimize

minimize 会将打包出来的bundle文件进行压缩和去除注释；

在开启Tree Shaking的情况下，在开启minimize

//webpack.config.js
...
 optimization:{
    usedExports:true,
    minimize:true
}
...

在没有开启的情况，如下图： 在开启的情况，如下图

并且文件的大小对比如下

minimize:false;
// asset bundle.js 2.76 KiB [emitted] (name: main);

minimize:true;
// asset bundle.js 381 bytes [emitted] [minimized] (name: main)
